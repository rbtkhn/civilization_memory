REVIEWER–BOOTSTRAP–MAP
Civilizational Memory Codex · Fast Orientation Map

Status: ACTIVE
Governed by: CMC 3.3
Last Updated: 2026-02-13

Purpose:
Help new reviewers and newly initialized models locate canonical governance
rules quickly, distinguish canonical from legacy-compatible patterns, and
avoid false-positive compliance findings.

────────────────────────────────────────────────────────────
I. LOAD ORDER (MINIMUM)
────────────────────────────────────────────────────────────
1) docs/governance/CMC–BOOTSTRAP.md
2) docs/governance/VERSION–MANIFEST.md
3) docs/governance/CIV–MEM–CORE.md (binding and mandatory sections)
4) .cursor/rules/cmc-*.mdc (mode, options, voice, STATE grounding)

────────────────────────────────────────────────────────────
II. CANONICAL PRECEDENCE
────────────────────────────────────────────────────────────
1) VERSION–MANIFEST (declares active governance version)
2) CIV–MEM–CORE (global operating law)
3) Templates (authoring/enforcement shape)
4) Cursor rules (runtime enforcement hints and guardrails)
5) Content files (MEM/CORE/SCHOLAR/STATE/DOCTRINE/ARC)

When statements conflict, higher-precedence source wins.

────────────────────────────────────────────────────────────
III. KNOWN LEGACY-COMPATIBLE PATTERNS
────────────────────────────────────────────────────────────
These are not automatic violations:
- Legacy MEM headers with `Governed by` or `Template Version Used`
- Legacy governance-version references inside older civilization files
- Older section labels that remain structurally equivalent

See: docs/governance/LEGACY–HEADER–COMPATIBILITY.md

────────────────────────────────────────────────────────────
IV. HIGH-RISK DRIFT HOTSPOTS
────────────────────────────────────────────────────────────
- Options menu word-count requirements across multiple rule files
- STATE Source Versions vs live source file headers
- MEM header/footer version parity
- Mandatory sections in MEM files (e.g., MEM CONNECTIONS)

────────────────────────────────────────────────────────────
V. MINIMUM REVIEW CHECKS
────────────────────────────────────────────────────────────
- Confirm mode contracts match BOOTSTRAP + cursor rules
- Confirm 8-slot A–H options policy consistency
- Confirm STATE safeguards (MEM scan, ARC coverage, contradiction surfacing)
- Confirm version-decoupling is respected (content vs governance version)
- Confirm sampled MEM files meet mandatory section requirements
