PROTOCOL–CONTRIBUTOR–TRIAGE
Civilizational Memory Codex · Quality and Bad-Faith Triage

Status: ACTIVE
Governed by: CMC 3.2
Last Updated: 2026-02-13

Purpose:
Provide a concrete triage flow for low-quality, non-compliant, or bad-faith
contributions while preserving CMC's contradiction-preserving principles.

────────────────────────────────────────────────────────────
I. TRIAGE CATEGORIES
────────────────────────────────────────────────────────────
- **T1 Structural Drift:** naming/schema/mandatory-section failures
- **T2 Governance Conflict:** mode/directionality/rule contradictions
- **T3 Evidence Integrity:** source loss, attribution breakage, quote misuse
- **T4 Behavioral Risk:** repeated bypassing of policy, adversarial edits

────────────────────────────────────────────────────────────
II. SEVERITY LEVELS
────────────────────────────────────────────────────────────
- **Minor:** local formatting/section labeling drift; low risk
- **Material:** governance mismatch with operational impact
- **Critical:** policy bypass, provenance failure, deliberate corruption

────────────────────────────────────────────────────────────
III. RESPONSE PLAYBOOK
────────────────────────────────────────────────────────────
1) Detect
   - CI failure, reviewer finding, or issue report.
2) Classify
   - Assign category (T1–T4) and severity (Minor/Material/Critical).
3) Contain
   - Block merge for Material/Critical.
4) Remediate
   - Request changes with concrete checklist.
5) Escalate (if recurring)
   - Maintainer review; temporary contributor restriction if needed.

────────────────────────────────────────────────────────────
IV. BAD-FAITH SIGNALS
────────────────────────────────────────────────────────────
Signals requiring escalation:
- deliberate removal of contradictory evidence
- repeated attempts to bypass mandatory checks
- intentional misattribution or fabricated source references
- repeated semantic relabeling to evade policy terms

────────────────────────────────────────────────────────────
V. ISSUE INTAKE
────────────────────────────────────────────────────────────
Use issue template `Quality/Governance Report` with:
- affected paths
- category + severity
- observed vs expected behavior
- reproducibility and evidence

────────────────────────────────────────────────────────────
VI. CLOSURE CRITERIA
────────────────────────────────────────────────────────────
Close when:
- CI checks pass
- reviewer checklist satisfied
- provenance/signing obligations met where required
- changelog updated if normative governance changed
