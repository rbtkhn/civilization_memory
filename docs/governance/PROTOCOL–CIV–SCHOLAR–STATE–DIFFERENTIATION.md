# PROTOCOL: Observation and Analysis of CIV–SCHOLAR / CIV–STATE Differentiation

**Classification:** Governance · Protocol  
**Status:** ACTIVE  
**Version:** 1.0  
**Last Updated:** February 2026  
**Purpose:** Provide a clear, systematic way to observe and analyze differentiation between different civilization SCHOLAR and STATE files.

---

## I. Purpose and Scope

Differentiation analysis answers:

- **Observation:** What does each file actually encode (structure, content, constraints)?
- **Analysis:** How do two or more civilizations differ along comparable dimensions?
- **Use:** Cross-civ comparison, gap detection, template compliance, cognitive-spiritual contrast.

This protocol defines **dimensions to observe**, **how to compare**, and **how to report**. It does not prescribe conclusions; it ensures consistent extraction and comparison so differentiation is reproducible.

**Scope:**

- **SCHOLAR:** CIV–SCHOLAR–[CIV] files (constraint grammar, RLLs, axioms, NCZ, phase, syntheses).
- **STATE:** CIV–STATE–[CIV] files (material options, stability indicators, decision-relevant history, cross-entity links).
- **Cross-file:** Optional joint run (e.g. “SCHOLAR vs STATE encoding of the same entity”).

**Exemplars:** REPORT–SCHOLAR–FRANCE–GERMANY–COGNITIVE–DIFFERENTIATION.md, REPORT–SCHOLAR–FRANCE–ANGLIA–COGNITIVE–DIFFERENTIATION.md.

## II. Observation Dimensions

### A. SCHOLAR Observation Dimensions

Extract and record the following for each CIV–SCHOLAR–[CIV] in scope. Use verbatim quotes where the dimension is a single declaration; use structured summary where the dimension is multi-part.

| # | Dimension | What to extract | Example (illustrative) |
|---|-----------|-----------------|-------------------------|
| S1 | **Self-conception** | One-sentence role (ledger, constraint grammar, etc.) | "accumulative learning ledger" vs "constraint grammar layer" |
| S2 | **Phase** | Phase I / Phase II / READ-ONLY / none | Phase I (Accumulation) vs Phase II (Constraint Grammar) |
| S3 | **Initial state & authority** | Starting knowledge (ZERO vs ACCUMULATED); CORE/DOCTRINE enabled or disabled | ZERO, CORE DISABLED vs ACCUMULATED, CORE ENABLED |
| S4 | **Axioms** | Count; one-line summary per axiom (theme: authority, legitimacy, rupture, limits, etc.) | 5 axioms (declaration, universalism, rupture…) vs 7 (discipline, stopping rules, trauma…) |
| S5 | **Negative Capability Zone (NCZ)** | Count; epistemic vs constraint (gaps to fill vs exclusions); resolution path type | CONTESTED + resolution path (ingest MEMs) vs INSUFFICIENT + counter-case required |
| S6 | **RLLs** | Bound count; proposed count; activation triggers present? | 0 bound, N proposed vs 11 bound, 5 proposed |
| S7 | **Synthesis → constraint** | Do syntheses become binding (RLLs) or remain frozen only? | Doctrines frozen, RLLs proposed vs Syntheses → bound RLLs |
| S8 | **Lock / write semantics** | READ-ONLY, APPEND-ONLY, no autonomous doctrine creation, etc. | As stated in file header or Section I/II |
| S9 | **Section coverage** | Which template sections are present and populated (IV Axioms, V RLLs, VI NCZ, session log, etc.) | List section IDs and brief fill status |

**Optional (deeper run):** Session-log density, OGE embedding, MIND profile embedding, doctrine count and naming.

### B. STATE Observation Dimensions

Extract and record the following for each CIV–STATE–[CIV] in scope.

| # | Dimension | What to extract | Example (illustrative) |
|---|-----------|-----------------|-------------------------|
| T1 | **Entity classification** | CIVILIZATION-STATE / STANDARD-STATE / FRAGMENTED-SOURCE / SUCCESSOR-STATE | Russia: CIVILIZATION-STATE; Germany: STANDARD-STATE |
| T2 | **Strategic position (III)** | Presence and balance of three perspectives (legitimacy, power, liability); tensions noted? | All three present; tensions preserved |
| T3 | **Material options (IV)** | Count; structure (grounding, assumptions, confidence, risks); accommodation/reversal option present? | 3–6 options; at least one accommodation/reversal if commitment decision |
| T4 | **Stability indicators (VI)** | Count; status (e.g. STRESSED, CRITICAL); threshold rules | 5 indicators; 2 STRESSED |
| T5 | **Opponent constraint (VI-B)** | Present or N/A; if present: opponent named, relative degradation framed | Present; opponent Ukraine; relative attrition tracked |
| T6 | **Decision-relevant history (VII)** | Pattern count; precedent MEMs; doctrine-audit/forward-projection subsections present? | 4 patterns; 2 with MEM precedent; optional subsections present |
| T7 | **Cross-entity links (VIII)** | Count; types (adversary, partner, comparison) | 3 entities; adversary 1, partner 2 |
| T8 | **Source versions (IX)** | SCHOLAR/CORE/DOCTRINE versions STATE was last synced to | SCHOLAR v3.3, CORE v2.1, DOCTRINE v3.4 |
| T9 | **State log / activity** | Entry count; last activity type (Decision Points, Stability Watch, Pattern Audit, etc.) | 14 entries; last: Decision Points, Geneva round |

**Optional (deeper run):** Option confidence distribution, discriminating evidence quality, sync recency.

### C. Cross-File (SCHOLAR vs STATE for Same Entity)

When comparing SCHOLAR and STATE for the **same** civilization:

| # | Dimension | What to observe |
|---|-----------|-----------------|
| X1 | **Constraint flow** | Do STATE material options reference SCHOLAR RLLs or patterns? |
| X2 | **Vocabulary alignment** | Same terms for legitimacy, power, liability? |
| X3 | **Decision-relevant history** | STATE Section VII patterns linked to SCHOLAR syntheses or MEM clusters? |
| X4 | **Gap** | Themes in SCHOLAR with no STATE counterpart (or vice versa)? |

## III. Analysis Types

Choose one (or combine) according to goal:

| Type | Description | Output |
|------|-------------|--------|
| **Pairwise contrast** | Two entities (e.g. Russia vs Germany). Extract S1–S9 and/or T1–T9 for both; compare dimension by dimension. | Comparison tables + executive summary of “distinct epistemes” or structural differences. |
| **N-way comparison** | Three or more entities. Same dimensions; fill a matrix (rows = dimensions, columns = entities). | Matrix + summary of clusters (e.g. “Phase I cluster: Francia, Anglia; Phase II: Germania, Russia”). |
| **Gap analysis** | One entity. Measure against template or against another entity’s “full” implementation. | List of missing or thin dimensions; recommendations. |
| **Cross-file (same entity)** | SCHOLAR vs STATE for one civ. Use X1–X4. | Short report: constraint flow, alignment, gaps. |

---

## IV. Procedure

1. **Scope the run**
   - Decide: SCHOLAR only, STATE only, or both.
   - Decide: pairwise (which two?), n-way (which set?), gap-only (which entity?), or cross-file (which entity?).

2. **Gather files**
   - Read each CIV–SCHOLAR–[CIV] and/or CIV–STATE–[CIV] in scope.
   - Use current versions; record version and date in the report.

3. **Extract**
   - For each file, fill the relevant dimension table (S1–S9 and/or T1–T9, and optionally X1–X4).
   - Prefer verbatim quotes for self-conception, phase, authority, and axiom/NCZ characterisation.

4. **Compare**
   - Align dimensions side by side (pairwise or n-way matrix).
   - Note: same template slot (e.g. “Authority” axiom) can encode different cognitive content (e.g. declaration vs discipline).

5. **Synthesise**
   - Write a short executive summary: what is the main differentiation? (e.g. “accumulation vs constraint grammar”, “epistemic NCZ vs modal NCZ”).
   - Preserve tension; do not resolve contradictions between civilizational encodings.

6. **Report**
   - Use the output format below (or the structure of REPORT–SCHOLAR–FRANCE–GERMANY as template).
   - Store in docs/governance with name e.g. REPORT–SCHOLAR–[CIV1]–[CIV2]–DIFFERENTIATION–[YYYY-MM-DD].md or REPORT–STATE–[CIV1]–[CIV2]–DIFFERENTIATION–[YYYY-MM-DD].md.

## V. Output Format

**Minimal report structure:**

```markdown
# Report: [SCHOLAR|STATE|SCHOLAR+STATE] Differentiation — [CIV1] vs [CIV2] [vs …]

**Classification:** Governance · Comparative Analysis
**Source:** CIV–SCHOLAR–[CIV1] vX, CIV–SCHOLAR–[CIV2] vY [, …]
**Date:** YYYY-MM-DD
**Protocol:** PROTOCOL–CIV–SCHOLAR–STATE–DIFFERENTIATION v1.0

## I. Executive Summary
[2–4 sentences: main differentiation; distinct epistemes or structural contrast.]

## II. Observation Summary
[Optional: single table with dimensions as rows, entities as columns.]

## III. Dimension-by-Dimension [or Section-by-Section]
[For each dimension: verbatim or summary for each entity; optional contrast table.]

## IV. Demonstration Tables
[Side-by-side comparison tables for key dimensions.]

## V. Conclusion / Use
[When to use this differentiation; cross-reference to SCHOLAR/STATE files.]
```

**Differentiation report exemplars:**

- REPORT–SCHOLAR–FRANCE–GERMANY–COGNITIVE–DIFFERENTIATION.md (SCHOLAR pairwise; cognitive-spiritual)
- REPORT–SCHOLAR–FRANCE–ANGLIA–COGNITIVE–DIFFERENTIATION.md (SCHOLAR pairwise; phase and lock)
