C.1 OPERATIONALIZABILITY
────────────────────────────────────────────────────────────

**Question:** 0001 states "synthesis forbidden—tension preserved." Does a *doctrine* that says "Carrhae-type events are multi-frame; no single-frame closure" add CORE/STATE utility? Or is it purely descriptive (Carrhae encoding divergence)?

**Analysis:**

- **Descriptive content:** 0001 records that Carrhae admits four non-synthesized readings (Keegan: civilised vs oriental war, encoding divergence; Mearsheimer: incentive structure; Barnes: liability; Mercouris: legitimacy). The tension table is a *description* of how the event was analyzed in LEARN, not a rule for future analysis.
- **Potential operational constraint:** "When analyzing Carrhae-type events (founding Roman–Persian battles, or encoding-divergence events), apply all four frames; no single-frame closure." That would constrain *how* we analyze a narrow class of events.
- **CORE utility:** CORE does not perform narrative or multi-frame analysis; it outputs diagnostics (DS, PAC, SGC, etc.). A "multi-frame for Carrhae-type" doctrine would not change CORE outputs.
- **STATE utility:** STATE might invoke historical parallels (e.g. "Carrhae as denial precedent"). A doctrine could require that when Carrhae or "Carrhae-type" is invoked, multiple frames are cited rather than a single reading. That is a weak constraint—STATE already cites MEMs and can cite multiple perspectives without a binding doctrine.
- **Overlap:** The general non-synthesis rule (Template) already forbids closure-seeking synthesis. Three-perspective analysis (Mercouris, Mearsheimer, Barnes) is already required. 0001 adds Keegan as a *fourth* frame and names Carrhae as the exemplar. So the incremental constraint is: "for Carrhae-type events, include Keegan/belief/encoding frame and preserve tension." That is narrow and partly descriptive (Carrhae encoding divergence).

**Decision:** **Primarily descriptive.** A doctrine would add a narrow operational constraint (multi-frame for "Carrhae-type" events) that overlaps with non-synthesis and three-perspective practice. CORE gains nothing; STATE gains at most a reminder to multi-frame when citing founding/encoding-divergence battles. The main value of 0001 is **pedagogical and archival**—it records that Carrhae was analyzed under four frames and tension was preserved. Operationalizing it as doctrine would add little beyond what is already required by the non-synthesis rule and optional four-frame practice (as in Exercise B).

C.2 MEM CONSISTENCY — REPEATABILITY VS UNIQUENESS
────────────────────────────────────────────────────────────

**Question:** Does MEM–PERSIA–WAR–CARRHAE support "four frames, tension preserved" as a *repeatable* pattern (other battles same treatment)? Or is Carrhae unique?

**MEM–PERSIA–WAR–CARRHAE (v1.6) summary:**
- **Purpose:** Carrhae as "archetypal denial victory" and "founding proof" that endurance doctrine works against the West.
- **Frames present in MEM:** Structural (VI: power distribution, incentive, misperception); civilizational/legitimacy (III, V: boundary, legitimacy encoded in refusal of Roman-backed king); Barnes (VII: who paid, who benefited). Keegan is *not* in the MEM text—Keegan was applied in LEARN (ENTRY 0002) via ENTRY 0001 (Keegan corpus) as external contrast ("civilised warfare" vs Persian denial).
- **Repeatability:** The MEM labels Carrhae as *archetypal* and *founding*—i.e. the first or defining instance of a type. Other battles in the WAR corpus (e.g. Thermopylae, Alexander, later Roman–Persian clashes) could in principle receive the same four-frame treatment. So the pattern is **repeatable as a type**: "founding or archetypal Roman–Persian / Persian-doctrine battles (or encoding-divergence events)." Carrhae is the **named exemplar**; it is not unique in the sense that no other event could be analyzed the same way, but it is unique as the *first* such analysis in SCHOLAR and as the battle that "inaugurates" the seven-century rivalry (per MEM and MEM–PERSIA–WAR–ROME).

**Result:** **Repeatable as a type.** The MEM supports that Carrhae is an exemplar of a class (founding/archetypal denial or encoding-divergence events). Other events in that class could get the same multi-frame, tension-preserved treatment. The MEM does not itself state "apply four frames to all such events"; that would be a doctrine we could add. So: repeatability supports a *possible* meta-constraint ("for events of type X, multi-frame required"); uniqueness is only that Carrhae is the first and most explicitly analyzed instance.

C.3 CONFLICT WITH NON-SYNTHESIS RULE
────────────────────────────────────────────────────────────

**Question:** A doctrine that says "preserve tension" is meta. Does it inadvertently freeze a single reading of Carrhae?

**Check:** 0001 does not state "the correct reading of Carrhae is X." It states that (1) Carrhae is analyzable through multiple frames, (2) frames produce different but complementary insights, (3) synthesis is forbidden—tension preserved. So the doctrine would be **meta**: "when analyzing Carrhae-type events, do not close on one frame; preserve tension." It does not prescribe which reading is true or primary. It prescribes *method* (multi-frame, no closure), not *content* (e.g. "Crassus was liable" or "structure explains it"). Therefore **no single-frame freeze**; the rule is meta only. It is consistent with the Template non-synthesis rule (preserve tensions; do not resolve contradictions).

**Result:** **No conflict.** A doctrine derived from 0001 would be a meta-rule only; it would not freeze a single reading of Carrhae.

────────────────────────────────────────────────────────────
RECOMMENDATION
────────────────────────────────────────────────────────────

**RETAIN UNBINDING.**

**Rationale:**
1. **Operationalizability (C.1):** 0001 is primarily descriptive—it records a multi-frame analysis of Carrhae and the decision to preserve tension. Turning it into doctrine would add only a narrow constraint (multi-frame for "Carrhae-type" events) that overlaps with the existing non-synthesis rule and three-perspective practice. CORE gains nothing; STATE gains at most a light reminder. The main value is pedagogical/archival.
2. **Repeatability (C.2):** The MEM supports repeatability *as a type* (founding/archetypal events), but the MEM does not demand that all such events receive four-frame treatment—that would be a new doctrine. So repeatability does not *require* binding; it only makes binding *possible* if we wanted a "Carrhae-type = multi-frame required" rule.
3. **No freeze (C.3):** Any doctrine would be meta only; no conflict with non-synthesis rule.
4. **Consistency with Exercise B:** We recommended RETAIN UNBINDING for 0003 (Khosrow four-frame). Binding 0001 (Carrhae four-frame) would create asymmetry: multi-frame required for Carrhae-type but not for overextension-type, unless we also bound 0003. Since we did not bind 0003, binding 0001 alone would add a single, narrow meta-rule for "founding/encoding-divergence" events without a broader multi-frame doctrine. That is optional and low yield.

**If the user wants a single meta-doctrine covering both:** Consider a single DOCTRINE 03 that says "When analyzing archetypal or founding events (e.g. Carrhae) or overextension cases (e.g. Khosrow II), all four analytical frames must be considered; no single-frame closure." That would subsume both 0001 and 0003 under one procedural doctrine. Exercise B recommended RETAIN for 0003; Exercise C recommends RETAIN for 0001. A combined doctrine remains optional.
