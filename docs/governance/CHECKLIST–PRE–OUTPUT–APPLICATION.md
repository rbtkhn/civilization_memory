CHECKLIST–PRE–OUTPUT–APPLICATION
Civilizational Memory Codex · Consolidated Pre-Output Checks

Status: ACTIVE
Governed by: CMC 3.4
Last Updated: February 2026

Purpose:
Run the applicable checklist below **before delivering** output that falls under any trigger. This document consolidates self-checks from cmc-arc-source-coverage, cmc-negative-claim-check, and cmc-gated-spiral-awareness so they are applied consistently. The cursor rules remain the authoritative specification; this file is the single run point.

────────────────────────────────────────────────────────────
I. STATE EVIDENCE / DOCTRINE CHECK (ARC SOURCE COVERAGE)
────────────────────────────────────────────────────────────

**Trigger:** About to present current-event evidence in STATE mode or in a doctrine check (Pattern Audit, Sync Protocol, Evidence Update, Stability Indicator, Decision Points, or free-form STATE analysis).

**Before delivering, confirm:**
- [ ] At least two ARC source categories consulted for the entity?
- [ ] At least one source from the entity's own official/institutional category (e.g. E/OFFICIAL or E/RESEARCH for Russia) where available?
- [ ] Every factual claim attributed with source name, ARC origin, and editorial note?
- [ ] Source contradictions surfaced (both claims with attribution, not silently resolved)?
- [ ] Evidentiary quality differences annotated where relevant (annotation only, not resolution)?

If any box is unchecked, complete the step before delivering.

Reference: cmc-arc-source-coverage.mdc; CIV–STATE–TEMPLATE SOURCE CONTRADICTION (BINDING).

────────────────────────────────────────────────────────────
II. ABSENCE CLAIM (NEGATIVE CLAIM CORPUS CHECK)
────────────────────────────────────────────────────────────

**Trigger:** About to assert that a civilisation lacks a quality, trait, tradition, or pattern (e.g. "X has no tradition of Y", "X's grammar lacks Y", "X has no concept of Y", "X cannot [do Y]", or softer variants).

**Before delivering, confirm:**
- [ ] Does the output contain any absence claim about a civilisation?
- [ ] If yes: was the relevant MEM corpus searched for counter-evidence (institutions and persons)?
- [ ] If counter-evidence found: was the claim reformulated (not merely softened)—e.g. "contains the category but has not sustained it" or "subordinate to [other pattern]"?
- [ ] If divergence/comparative analysis: was the check applied to each absence claim for each civilisation?

If any box is unchecked, complete the step before delivering.

Reference: cmc-negative-claim-check.mdc.

────────────────────────────────────────────────────────────
III. DOCTRINE-CHANGE PROPOSAL (GATED SPIRAL AWARENESS)
────────────────────────────────────────────────────────────

**Trigger:** About to propose a new or modified doctrine (doctrine addition, refinement, or encoding in CIV–DOCTRINE or SCHOLAR-derived doctrine proposal).

**Before delivering, confirm:**
- [ ] Have I noted which doctrines governed the analysis that produced the finding?
- [ ] Have I stated whether this finding is frame-consistent (extends existing doctrines) or frame-challenging (contradicts or complicates them)?
- [ ] Have I included the required transparency sentence?  
  — Frame-consistent: "This finding was produced by analysis governed by [DOCTRINE X / Y]. The questions that generated it were shaped by those doctrines. The finding is consistent with the existing analytical frame; it does not challenge it."  
  — Frame-challenging: "This finding was produced by analysis governed by [DOCTRINE X] but contradicts or complicates [DOCTRINE Y]. The finding may represent genuine discovery rather than frame-confirmation."

If any box is unchecked, complete the step before delivering.

Reference: cmc-gated-spiral-awareness.mdc.

────────────────────────────────────────────────────────────
IV. WHEN MULTIPLE TRIGGERS APPLY
────────────────────────────────────────────────────────────

Run all applicable sections (I, II, III) before delivering. Order: run I if presenting STATE evidence; run II if making any absence claim; run III if proposing doctrine change.
