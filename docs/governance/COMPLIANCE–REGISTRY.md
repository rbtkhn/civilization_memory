────────────────────────────────────────────────────────────
I. PURPOSE
────────────────────────────────────────────────────────────
This registry tracks compliance status for all MEM files.

Previously, each MEM file declared its own compliance status in headers:
• "20% COMPLIANT"
• "COMPLIANT (CIV–MEM–TEMPLATE v3.0)"
• etc.

Under CMC 3.1 Version Decoupling:
• Compliance status is tracked HERE, not in individual files
• MEM headers are simplified (no governance declarations)
• Validation results are centralized and queryable

────────────────────────────────────────────────────────────
II. COMPLIANCE CRITERIA (CMC 3.1)
────────────────────────────────────────────────────────────
A MEM is COMPLIANT if:

STRUCTURAL:
• ≥8 analytical sections
• MEM CONNECTIONS section present
• MEM BIBLIOGRAPHY section present
• MEM INGEST BOOTSTRAP section present

EVIDENTIARY:
• ≥20% verbatim quotes by word count
• ARC sources properly cited
• 4-era coverage (for GEO–MEMs)

CONNECTIONS:
• ≥10 same-civilization connections
• ≥2 GEO–MEM connections

STATUS LEVELS:
• COMPLIANT — Meets all criteria
• PARTIAL — Meets most criteria, minor gaps
• LEGACY — Pre-CMC 3.1 format, content valid
• NEEDS_UPGRADE — Structural issues requiring attention

IX. VALIDATION PROCESS
────────────────────────────────────────────────────────────
Compliance is validated:

AUTOMATICALLY (cmc-console):
• Word count check
• Section presence check
• Connection count check
• Quote percentage estimate

MANUALLY (AUDIT function):
• ARC source quality
• Quote relevance
• Connection meaningfulness
• Analytical depth

FREQUENCY:
• Major batches: After significant MEM creation/upgrade
• Spot checks: During LEARN sessions
• Full audit: Quarterly (recommended)

────────────────────────────────────────────────────────────
X. MAINTENANCE
────────────────────────────────────────────────────────────
This registry is updated:
• When MEMs are created or significantly modified
• When compliance criteria change
• After batch validation runs

Update format:
• Increment counts in civilization sections
• Add specific files to PARTIAL/NEEDS_UPGRADE lists
• Update "Last Validated" dates
