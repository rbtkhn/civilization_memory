CMC–MEM–PROTOCOL — V1.0
Civilizational Memory Codex (CMC)
Memory Object Protocol · Canonical

Status: ACTIVE · CANONICAL
Applies To: All files under `CIVILIZATION/*/MEMORY/`
Lock Level: STRUCTURAL
Supersedes: All pre-CMC historical shard conventions (conceptual lineage only)

────────────────────────────────────────────────────────────
I. PURPOSE & ONTOLOGY
────────────────────────────────────────────────────────────

The Civilizational Memory Codex treats history as **memory**, not narrative.

A Memory is:
• a retained civilizational experience
• metabolized rather than erased
• selectively activated
• structurally consequential

Memory conditions behavior.
Behavior produces strategy.

Memory objects exist to encode **how civilizations remember**.

────────────────────────────────────────────────────────────
II. WHAT A MEMORY IS (DEFINITION)
────────────────────────────────────────────────────────────

A Memory object is a modular, promotable historical unit that:
• represents an event, dynasty, person, institution, or technique
• encodes lessons without verdict compression
• is consumable by Civilization Engines
• preserves causality, not storytelling

Memories do not argue.
They **inform constraint**.

────────────────────────────────────────────────────────────
III. MEMORY NAMING & FILE IDENTITY
────────────────────────────────────────────────────────────

All Memory files MUST be named using:


Examples:
• CHINA_MEMORY_SONG_TECHNOCRATIC_LEGITIMACY.md
• GERMANIA_MEMORY_PRUSSIAN_BUREAUCRATIC_DISCIPLINE.md

Each file MUST include:
• Civilization
• Era (Ancient / Medieval / Colonial / Industrial / Modern)
• Memory Status
• Memory Type (event / dynasty / person / technique / trauma)
• Engine Compatibility (CMC version)

────────────────────────────────────────────────────────────
IV. MEMORY PROMOTION STATES
────────────────────────────────────────────────────────────

Every Memory MUST declare one status:

• DRAFT — exploratory, incomplete, expandable
• CANDIDATE — citation-bearing, internally coherent
• FINAL — analytically stable, engine-consumable
• LOCKED — archival, no modification without version bump

Promotion requires explicit declaration.
Silent promotion is prohibited.

────────────────────────────────────────────────────────────
V. REQUIRED MEMORY STRUCTURE
────────────────────────────────────────────────────────────

Every Memory MUST contain the following sections, in order:

I.   Memory Statement (why this memory exists)
II.  Historical Context
III. Mechanism of Memory Formation
IV.  What Is Remembered
V.   What Is Suppressed or Minimized
VI.  Civilizational Logic Extracted
VII. Engine Linkages
VIII.Primary Source Anchors
IX.  Promotion Notes

Sections may be expanded but not removed or reordered.

────────────────────────────────────────────────────────────
VI. CITATION & SOURCE DISCIPLINE
────────────────────────────────────────────────────────────

Memory objects are Scholar-mode native.

Each Memory SHOULD include:
• primary-source quotations when available
• explicit source identification
• quotation density assessment
• citation sufficiency scoring (0–5)

Absence of sources MUST be acknowledged explicitly.
Inference must be labeled as such.

────────────────────────────────────────────────────────────
VII. MEMORY VS ENGINE SEPARATION RULE
────────────────────────────────────────────────────────────

Memory objects:
• may contain narrative
• may contain quotation
• may contain historical exposition

Civilization Engines:
• MUST NOT contain extended narrative
• MUST NOT substitute memory content
• MAY reference Memory titles only

Engines consume memories.
They do not overwrite them.

────────────────────────────────────────────────────────────
VIII. TEMPORAL STACKING RULE
────────────────────────────────────────────────────────────

Multiple Memories MAY coexist for the same era.

Contradictory Memories:
• are allowed
• must be explicitly labeled
• indicate contested or layered recall

Civilizations may remember inconsistently.
This is a feature, not a bug.

────────────────────────────────────────────────────────────
IX. FAILURE & TRAUMA HANDLING
────────────────────────────────────────────────────────────

Defeats, collapses, and occupations MUST be treated as:
• memory amplifiers
• constraint generators
• long-duration modifiers

Failure does not invalidate civilization.
It reshapes memory weighting.

────────────────────────────────────────────────────────────
X. SCHOLAR MODE REQUIREMENT
────────────────────────────────────────────────────────────

All Memory development occurs under Scholar Mode.

Memory creation forbids:
• predictive forecasting
• verdict compression
• adversarial simulation
• policy recommendation

The purpose of Memory is preservation and clarity,
not decision-making.

────────────────────────────────────────────────────────────
XI. CANONICAL AUTHORITY
────────────────────────────────────────────────────────────

This protocol:
• governs all Memory objects in the CMC
• is binding across civilizations
• overrides informal or legacy shard rules
• may be revised only via versioned replacement

If a Memory violates this protocol,
it is not canonical.
