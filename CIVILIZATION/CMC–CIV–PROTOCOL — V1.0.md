────────────────────────────────────────────────────────────
I. PURPOSE & ONTOLOGY
────────────────────────────────────────────────────────────

The Civilizational Memory Codex (CMC) models civilizations as
**continuity systems governed by accumulated memory**, not as
nation-states, regimes, or ideological projects.

A Civilization Engine is:
• a systems model
• memory-driven
• non-narrative
• analytically bounded
• structurally comparable across civilizations

History supplies **memory**.
Memory conditions **behavior**.
Behavior produces **strategy**.

The CMC encodes this chain.

────────────────────────────────────────────────────────────
II. CIVILIZATION ENGINES (DEFINITION)
────────────────────────────────────────────────────────────

A Civilization Engine is a canonical analytical instrument that:
• encodes civilizational logic
• consumes historical memory objects
• produces bounded analytical outputs
• forbids silent assumptions
• resists restoration fantasy

Engines do not predict.
Engines classify.

────────────────────────────────────────────────────────────
V. MEMORY PRIMACY RULE
────────────────────────────────────────────────────────────

Civilizations do not “reset.”
They **remember**.

All engines MUST assume:
• defeat becomes memory
• trauma becomes constraint
• success becomes habit
• institutions outlive regimes

Historical content belongs in **Memory objects**, not engine cores.

────────────────────────────────────────────────────────────
VI. MEMORY OBJECTS (CONSUMPTION RULE)
────────────────────────────────────────────────────────────

Memory objects:
• are modular historical units
• may represent dynasties, events, people, or techniques
• are Scholar-mode native
• are promotable (Draft → Candidate → Final → Locked)

Engines may reference memories but MUST NOT
embed full historical narratives internally.

────────────────────────────────────────────────────────────
VII. OUTPUT DISCIPLINE (MANDATORY)
────────────────────────────────────────────────────────────

All analytical layers MUST define:
• bounded categorical outputs
• finite vocabularies
• falsifiable states

Prohibited:
• free-text verdicts
• probabilistic hedging
• teleological claims
• moral judgment

The Mandatory Verdict Block (Section XX) is required
in all engines and is authoritative.

────────────────────────────────────────────────────────────
VIII. FAILURE & IRREVERSIBILITY REQUIREMENT
────────────────────────────────────────────────────────────

Every engine MUST include:
• a Split-Axis Failure model
• an Irreversibility Grid
• a Restoration Invalidation Rule

If restoration is invalidated,
“return-to-normal” analysis is prohibited.

────────────────────────────────────────────────────────────
IX. SCHOLAR MODE (EXCEPTION SPACE)
────────────────────────────────────────────────────────────

Scholar Mode:
• permits long-form exposition
• permits primary-source quotation
• permits citation density scoring
• permits memory development

Scholar Mode forbids:
• forced verdict compression
• simulation pressure
• adversarial framing

Scholar Mode exists to **build memory**, not to decide outcomes.

────────────────────────────────────────────────────────────
X. LOCK LEVELS
────────────────────────────────────────────────────────────

STRUCTURAL LOCK:
• section order frozen
• outputs retained
• content expandable

FULL LOCK:
• no changes without version bump
• reserved for archived or deprecated engines

────────────────────────────────────────────────────────────
XI. CANONICAL AUTHORITY
────────────────────────────────────────────────────────────

This protocol:
• governs all CMC civilization engines
• is binding across the repository
• overrides all informal conventions
• may be revised only by versioned replacement

Silence does not imply permission.
If it is not specified, it is prohibited.
