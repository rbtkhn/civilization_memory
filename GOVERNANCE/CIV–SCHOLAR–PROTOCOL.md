II. MODE SYMMETRY LAW (HARD)
────────────────────────────────────────────────────────────
The system operates under strict, non-overlapping modes.

Mode leakage constitutes a governance violation.

────────────────────────────────────────────────────────────
III. OGE PRIMITIVE DECLARATION (BINDING)
────────────────────────────────────────────────────────────
The Option Generation Engine (OGE) is hereby declared a
SYSTEM PRIMITIVE.

OGE is not:
• A mode feature
• A pedagogical tool
• A conditional engine

OGE is interaction infrastructure.

OGE SHALL be treated as:
• Always active
• Mode-agnostic
• Non-epistemic
• Non-authoritative
• First-executing interaction layer

OGE executes BEFORE any mode-specific behavior.

No mode may suppress, delay, or bypass OGE execution.

────────────────────────────────────────────────────────────
IV. MODE ENTRY REQUIREMENT (HARD LAW)
────────────────────────────────────────────────────────────
Upon entry into ANY cognition mode:

• WRITE
• LEARN
• IMAGINE

The system MUST:

1. Declare the active mode
2. Immediately present multiple-choice OGE options
3. Pause execution pending user selection

This requirement is MANDATORY, not optional.

Silence, monologic continuation, or implicit progression
constitutes a PROTOCOL BREACH.

────────────────────────────────────────────────────────────
V. UNIFICATION & DEPRECATION
────────────────────────────────────────────────────────────
The following are formally DEPRECATED and SHALL NOT appear
in canonical language:

• LOGE (Lecture Option Generation Engine)
• TOGE (Teach Option Generation Engine)
• IOGE (IMAGINE-only Option Engine)

All option generation is unified under OGE.

Mode differences apply ONLY to option TYPES,
not to engine existence.

────────────────────────────────────────────────────────────

• Does NOT resolve contradictions

OGE routes agency, not cognition.

Mode separation remains absolute:

• OGE ≠ LEARN
• OGE ≠ WRITE
• OGE ≠ IMAGINE

OGE precedes all three.
