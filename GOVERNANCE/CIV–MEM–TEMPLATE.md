────────────────────────────────────────────────────────────
I. PURPOSE & AUTHORITY
────────────────────────────────────────────────────────────
This file defines the mandatory structure, evidentiary standards,
scholarly constraints, behavioral rules, and ingest behavior for all
MEM (Civilizational Memory) files within the Civilizational Memory Codex.

Core principle:
A MEM file must function as a self-contained historical intelligence
object—capable of activating structured inquiry even when ingested
alone into a new conversation.

MEM authoring is a jurisdictional act.

Authority Flow (NON-REVERSIBLE):
CIV–MEM–CORE → CIV–MEM–TEMPLATE → MEM Files → SCHOLAR / CORE / GAME / LLM

No MEM file is valid unless compliant with this template AND the
applicable ARC–[CIV] canon at the time of locking.

────────────────────────────────────────────────────────────
II. FILE IDENTITY & METADATA (MANDATORY)
────────────────────────────────────────────────────────────
Every MEM file MUST begin with the following metadata block,
in this exact order:

• Filename (canonical)
• Repository reference
• Status (ACTIVE / LOCKED / DRAFT)
• Version (semantic)
• Upgrade Type (if applicable)
• Civilization
• Regime / Mode (if applicable)
• Subject
• Dates (BC / AD only; no BCE / CE)
• Class: MEM (Memory)
• Last Update
• Wordcount (verified)

Omission, mis-ordering, or unverifiable wordcount INVALIDATES the MEM.

────────────────────────────────────────────────────────────
III. MEMORY PURPOSE & SCOPE (REQUIRED)
────────────────────────────────────────────────────────────
Each MEM file MUST explicitly state:

• What is being preserved
• Why it matters civilizationally
• What function it serves in memory
• What it explicitly does NOT resolve

MANDATORY CLAUSE (verbatim):

“Contradictions are preserved without synthesis.”

────────────────────────────────────────────────────────────
IV. STRUCTURAL CONTENT RULES (HARD)
────────────────────────────────────────────────────────────
• Minimum 8 numbered analytical sections
• Analytical, not narrative
• No teleology
• No retrospective inevitability
• No moral adjudication

Permitted:
• Structural tension
• Ambiguity
• Constraint recognition

────────────────────────────────────────────────────────────
V. SOURCE QUOTATION REQUIREMENTS (ARC-CATEGORY HARDENED)
────────────────────────────────────────────────────────────
Each MEM file MUST include verbatim quotations drawn exclusively from
the civilization's ARC–[CIV] canon.

All quotations MUST:
• Be embedded within analytical sections
• Be attributed (author + work)
• Support a specific analytical claim

A) ANCIENT
• ≥3 quotations
• Aggregate ≥75 words

B) MEDIEVAL (when applicable)
• ≥2 quotations (when medieval continuity is relevant)
• EACH ≥25 words

C) EARLY MODERN
• ≥2 quotations
• EACH ≥25 words

D) MODERN
• ≥2 quotations
• Fair use only

Failure INVALIDATES the MEM.

────────────────────────────────────────────────────────────
VI. SECONDARY SOURCE ENGAGEMENT STANDARDS
────────────────────────────────────────────────────────────
Secondary sources MUST be engaged analytically.
Citation without engagement is INVALID.

────────────────────────────────────────────────────────────
VII. MEM CONNECTION REQUIREMENTS
────────────────────────────────────────────────────────────
Each MEM MUST reference:
• ≥10 other MEM files
• All connections must be to MEM files from the same civilization
• ≥2 connections must be to GEO (geography) MEM files
Each reference MUST explain structural relevance.

────────────────────────────────────────────────────────────
XI. ARC VERSION PINNING (NEW · HARD)
────────────────────────────────────────────────────────────
Every MEM file MUST declare, in metadata or Section XII:

• ARC–[CIV] version used for compliance (e.g., ARC–ROME v1.8)

Ingestion MUST verify parity with:
• CIV–CORE–[CIV]
• CIV–SCHOLAR–[CIV]

Version mismatch triggers:
• Compliance failure
• Default Option 8 (Audit)

────────────────────────────────────────────────────────────
XII. ARC COMPLIANCE PREFLIGHT DELEGATION (NEW)
────────────────────────────────────────────────────────────
ARC compliance checking is delegated to a PRE-INGEST PREFLIGHT layer.

This layer verifies:
• Author admissibility
• Category completeness
• Quotation counts and lengths
• ARC version parity

MEM files:
• DO NOT self-validate
• DO NOT perform enforcement logic
• ARE BLOCKED if preflight fails

────────────────────────────────────────────────────────────
XIII. WORDCOUNT VERIFICATION RULE (NEW)
────────────────────────────────────────────────────────────
Wordcount is a LOCKED metadata field.

Verification authority:
• Compliance Engine / Preflight only

Rules:
• Content edits require unlock
• Metadata-only corrections permitted IF:
  – Content hash unchanged
  – Correction logged
  – Version unchanged

Unverified wordcount INVALIDATES canonical lock.
